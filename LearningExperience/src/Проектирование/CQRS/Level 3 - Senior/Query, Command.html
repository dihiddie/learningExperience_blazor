Сommand / Команда

Во-первых, команда запуска - единственный способ изменить состояние нашей системы. Команды отвечают за внесение всех изменений в систему. Если команды не было, состояние системы остается неизменным! Команда не должна возвращать никакого значения. Я реализую его как пару классов: Command и CommandHandler. Команда - это простой объект, который используется CommandHandler в качестве входного значения (параметров) для некоторой операции, которую он представляет. В моем видении команда просто вызывает определенные операции в доменной модели (не обязательно по одной операции на команду).

Query / Запрос

Аналогично, запрос является операцией чтения. Он считывает состояние системы, фильтрует, агрегирует и преобразует данные, чтобы доставить их в наиболее полезном формате. Он может быть выполнен несколько раз и не повлияет на состояние системы. Я использовал для их реализации один класс с некоторым методом Execute (…), но теперь я думаю, что разделение на Query и QueryHandler / QueryExecutor может быть полезным.
