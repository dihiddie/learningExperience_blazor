Блокировки уровня таблицы

  • разделяемая (shared, S);

    • Разделяемая блокировка резервирует ресурс (страницу или строку) только для чтения. Другие процессы не могут изменять заблокированный ресурс, пока блокировка не будет снята. Другие процессы не могут изменять заблокированный ресурс, пока блокировка не будет снята. С другой стороны, отдельные процессы могут устанавливать разделяемую блокировку для этого ресурса в то же самое время, т. е. другие процессы могут читать ресурс, заблокированный при помощи разделяемой блокировки.

  • монопольная (exclusive, X);

    • Когда происходит добавление, обновление или удаление строк, то SQL Server устанавливает монопольную блокировку (X) на строку и удерживает ее до окончания транзакции. Эта блокировка защищает объект (например, строку) от изменения другими сессиями. Важный момент - монопольные блокировки (X) не снимаются до окончания транзакции независимо от уровня изоляции. Чем дольше длится транзакция, тем дольше сохраняются монопольные блокировки (X).

  • разделяемая с намерением (intent shared, IS);

  • монопольная с намерением (intent exclusive, IX);

    • Монопольные блокировки с намерением (IX) всего-лишь показывают, что есть другие монопольные блокировки, но уже на других уровнях (ниже уровня текущей блокировки). То есть, что транзакция намеревается блокировать следующий нижележащий в иерархии объектов базы данных ресурс. Поэтому блокировки с намерением помещаются в иерархии объектов на один уровень выше того объекта, который этот процесс намеревается заблокировать.

  • разделяемая с монопольным намерением (shared with intent exclusive, SIX)

Блокировка уровня строк/страниц

  • Разделяемая блокировка (shared lock)

    • Разделяемая блокировка резервирует ресурс (страницу или строку) только для чтения. Другие процессы не могут изменять заблокированный ресурс, пока блокировка не будет снята. Другие процессы не могут изменять заблокированный ресурс, пока блокировка не будет снята. С другой стороны, отдельные процессы могут устанавливать разделяемую блокировку для этого ресурса в то же самое время, т. е. другие процессы могут читать ресурс, заблокированный при помощи разделяемой блокировки.

  • Монопольная блокировка (exclusive lock)

    • Исключительная блокировка резервирует страницу или строку для исключительного использования в одной транзакции. Она используется для операторов DML (insert, update или delete), которые изменяют ресурс.

  • Блокировка обновления (update lock)

    • SQL Server устанавливает этот тип блокировок при поиске строк, которые необходимо обновить. После того, как блокировка обновления (U) установлена, SQL Server считывает строку и проверяет, нужно ли ее обновить, сверив значения строки с условиями в запросе. Если строка удовлетворяет условиям, то блокировка обновления (U) меняется на монопольную блокировку (X), затем строка модифицируется. В противном случае блокировка обновления (U) снимается.

WITH (NOLOCK)

Подсказка таблицы WITH (NOLOCK) используется для переопределения уровня изоляции транзакции по умолчанию для таблицы, позволяя пользователю получать данные, не подвергаясь влиянию блокировок, на запрошенные данные, из-за другого процесса, который его меняет. Таким образом, запрос будет занимать меньше памяти для удержания блокировок этих данных. Кроме того, не будет возникать тупиковой ситуации с запросами, которые запрашивают те же данные из этой таблицы, что обеспечивает более высокий уровень параллелизма из-за меньшей занимаемой площади. Другими словами, подсказка таблицы WITH (NOLOCK) извлекает строки, не ожидая, пока другие запросы, которые читают или изменяют те же данные, завершат свою обработку.
