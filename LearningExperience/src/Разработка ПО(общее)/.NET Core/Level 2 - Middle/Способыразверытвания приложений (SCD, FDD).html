Развертывание приложений с использование .NET CORE CLI

FDD (Framework-Dependent Deployment)

При FDD создается кроссплатформенный файл .dll, в котором используется локально установленная среда выполнения .NET Core.

Это дефолтная модель развертывания для команды dotnet publish в .NET Core SDK 2.x CLI

Когда вы публикуете свое приложение как FDD, файл <PROJECT-NAME> .dll создается в папке ./bin/<BUILD-CONFIGURATION>/<TFM>/publish/. Чтобы запустить приложение, перейдите в папку вывода и используйте команду dotnet <PROJECT-NAME> .dll

Получается, что ваше приложение настроена на определенную версию нет кор. И эта версия должна быть на любой машине, куда устанавливается ваше приложение

FDE (Framework-Dependent Executable)

При FDE создается исполняемый файл, зависящий от платформы, который использует локально установленную среду выполнения .NET Core.

Это дефолтная модель развертывания для команды dotnet publish в .NET Core SDK 3.x CLI

В этом режиме создается исполняемый хост, зависящий от платформы, для размещения вашего кроссплатформенного приложения. Этот режим похож на FDD, поскольку для FDD требуется хост в виде команды dotnet. Имя исполняемого файла хоста зависит от платформы и называется чем-то похожим на <PROJECT-FILE> .exe. Вы можете запустить этот исполняемый файл напрямую вместо вызова dotnet <PROJECT-FILE> .dll, что по-прежнему является приемлемым способом запуска приложения.

SCD (Self-Contained Deployment

При SCD создается исполняемый файл для конкретной платформы и включает локальную копию среды выполнения .NET Core.

Когда вы публикуете SCD, .NET Core SDK создает исполняемый файл для конкретной платформы. Публикация SCD включает все необходимые файлы .NET Core для запуска вашего приложения, но не включает собственные зависимости .NET Core. Эти зависимости должны присутствовать в системе до запуска приложения.
