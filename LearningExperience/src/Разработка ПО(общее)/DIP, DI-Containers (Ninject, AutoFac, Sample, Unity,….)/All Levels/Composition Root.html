Composition root это единственное место в приложение, где все модули составлены вместе. Грубо говоря, где разрешаются все зависимости

Составлять граф объектов следует как можно ближе к точке запуска. Composition Root может быть только у приложений, библиотеки и фреймворки не должны его содержать

На практике, это значит, что вы должны сконфигурировать контейнер в корне своего приложения

  • В десктопе - Application.OnStartup метод
  • В asp.net (включая mvc) - Global.asax
  • В WCF - ServiceHostFactory
  • Приложения ASP.NET Core содержат файл Startup.cs который является отправной точкой приложения, позволяющей настроить DI. Настраивается DI в методе ConfigureServices.
