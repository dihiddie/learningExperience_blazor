Entity Framework Core не поддерживает выполнение нескольких параллельных операций в одном экземпляре DbContext. Это включает как параллельное выполнение асинхронных запросов, так и любое явное одновременное использование из нескольких потоков. Поэтому всегда ожидайте асинхронных вызовов немедленно или используйте отдельные экземпляры DbContext для операций, которые выполняются параллельно.

Чтобы все было паралелльно надо:

  • Все асинхронные методы ef core awaitить незамедлительно
  • AddDbContext создает типы DbContext по умолчанию как scoped (один на клиента). Таким образом, доступ до инстансов dbContext никогда не будут одновременным
