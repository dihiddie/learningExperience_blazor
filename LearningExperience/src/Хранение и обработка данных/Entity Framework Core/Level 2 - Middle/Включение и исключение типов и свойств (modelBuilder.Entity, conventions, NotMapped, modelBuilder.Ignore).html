• modelBuilder.Entity - задание свойств энтити контекста
  • modelBuilder.Ignore - указывает, что энтити не должно быть смаплено в таблицу бд
  • notMapped атрибут - эквивалент modelBuilder.Ignore, указывает, что сущность или свойство не должно быть смаплено в бд
  • convensions (соглашения) - это набор правил, жестко встроенных в Entity Framework Core, которые определяют, как модель будет отображаться в схеме базы данных.
    • primary key - если свойство называется Id /EntityNameId, оно считается первичным
    • foreign key - тот же тип данных, что и первичный ключ и есть определенные паттерны ддя его наименования
    • backing fields - при соблюдении паттернов наименования, ef core будет считать поле к свойству как "поддерживающее"
    • table - энтити будет смаплена в таблицу бд при условии, что она имеет такое же название, как и указанное в dbset (сравнение работает и если одно в ед. числе, а другое во множественном)
    • schema - объект мапится в схему dbo по-умолчанию
    • columns - мапятся по тому же принципу одинаковости названия
    • indexes - всегда создаются на внешние и альтернативные ключи
