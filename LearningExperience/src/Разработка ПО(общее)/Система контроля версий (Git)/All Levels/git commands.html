git clone

Clone a repository into a new directory

git branch

Команда git branch делает несколько больше, чем просто создаёт и удаляет ветки. При запуске без параметров, вы получите простой список имеющихся у вас веток:

Чтобы посмотреть последний коммит на каждой из веток, выполните команду git branch -v:

Опции --merged и --no-merged могут отфильтровать этот список для вывода только тех веток, которые слиты или ещё не слиты в текущую ветку. Чтобы посмотреть те ветки, которые вы уже слили с текущей, можете выполнить команду git branch --merged:

git commit

Record changes to the repository

git fetch

git собирает все коммиты из целевой ветки, которых нет в текущей ветке, и сохраняет их в локальном репозитории. Однако он не сливает их в текущую ветку. Это особенно полезно, если вам нужно постоянно обновлять свой репозиторий, но вы работаете над функциональностью, неправильная реализация которой может негативно сказаться на проекте в целом.

Знает, что есть изменения, показывает их, но не стягивает их себе

git push

Update remote refs along with associated objects

git pull
git merge
git remote

Для того, чтобы просмотреть список настроенных удалённых репозиториев, вы можете запустить команду git remote. Она выведет названия доступных удалённых репозиториев. Если вы клонировали репозиторий, то увидите как минимум origin — имя по умолчанию, которое Git даёт серверу, с которого производилось клонирование:

git init

Create an empty Git repository or reinitialize an existing one

git revert

Revert some existing commits

git reset (soft, mixed, hard)

Reset current HEAD to the specified state

git-flow feature/release/hotfix

Clone a repository into a new directory

git cherry-pick

Команда git cherry-pick используется для того чтобы взять изменения, внесённые каким-либо коммитом, и попытаться применить их заново в виде нового коммита наверху текущей ветки. Это может оказаться полезным чтобы забрать парочку коммитов из другой ветки без полного слияния с той веткой.

git rebase

С помощью команды rebase вы можете взять все коммиты из одной ветки и в том же порядке применить их к другой ветке

git tag
git rm

Удаляет файл. Подробнее: https://www.atlassian.com/ru/git/tutorials/undoing-changes/git-rm
