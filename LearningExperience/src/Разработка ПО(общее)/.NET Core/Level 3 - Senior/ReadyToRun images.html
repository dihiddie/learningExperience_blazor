Можно улучшить время старта приложения (привет, jit-компилятор!) скомпилировав сборки приложения в формате ReadyToRun (R2R). R2R есть форма AOT компиляции.

Бинарники R2R улучшают производительность запуска за счет сокращения объема работы, которую JIT-компилятор должен выполнять при загрузке вашего приложения. Бинарники содержат нативный код, похожий на тот, что поставляет JIT-компилятор. Однако двоичные файлы R2R больше, поскольку они содержат как код промежуточного языка (IL), который все еще необходим для некоторых сценариев, так и собственную версию того же кода. R2R доступен только при публикации автономного приложения, ориентированного на определенные среды выполнения (RID), такие как Linux x64 или Windows x64.

Чтобы скомпилить свои приложения как ReadyToRun:

  • В csproj добавить <PublishReadyToRun> 
    • 
    <PropertyGroup>
        <PublishReadyToRun>true</PublishReadyToRun>
    </PropertyGroup>
  • Опубликовать self-contained приложение
    • dotnet publish -c Release -r win-x64 --self-contained
