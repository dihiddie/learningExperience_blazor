Можно использовать методы типа TryParse, чтобы, не используя try...catch можно обработать возможную исключительную ситуацию.

Фильтры исключений when

    catch (DivideByZeroException) when (y == 0 && x == 0)
    {
        // exception сработает только при соблюдении условия y == 0 && x == 0
        Console.WriteLine("y не должен быть равен 0");
    }

Класс Exception

Базовым для всех типов исключений является тип Exception. Этот тип определяет ряд свойств, с помощью которых можно получить информацию об исключении.

  • InnerException: хранит информацию об исключении, которое послужило причиной текущего исключения;
  • Message: хранит сообщение об исключении, текст ошибки;
  • Source: хранит имя объекта или сборки, которое вызвало исключение;
  • StackTrace: возвращает строковое представление стека вызовов, которые привели к возникновению исключения;
  • TargetSite: возвращает метод, в котором и было вызвано исключение.

Лучшие методики описаны тут

  • Использование блоков try/catch/finally для восстановления после ошибок или высвобождения ресурсов
  • Выдача исключений вместо возврата кода ошибки
  • Использование предопределенных типов исключений .NET (Создавайте новый класс исключений, только если предопределенное исключение не подходит)
  • Завершайте имена классов исключений словом Exception
  • Включение трех конструкторов в пользовательские классы исключений
  • Использование грамматически правильных сообщений об ошибке
