Пессимистический параллелизм включает в себя блокировку записей базы данных, чтобы другие пользователи не могли получить к ним доступ / изменить их до тех пор, пока блокировка не будет снята, подобно тому, как два пользователя пытаются открыть один и тот же файл в общей сетевой папке. Однако возможность блокировки записей поддерживается не всеми базами данных и может быть сложной для программирования, а также потребовать больших ресурсов. Это просто непрактично в автономных сценариях, таких как веб-приложения. Entity Framework Core не поддерживает пессимистичный контроль параллелизма.
