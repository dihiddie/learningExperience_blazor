Многоуровневая компиляция (Tiered compilation)

Включена по умолчанию с Net Core 3.0. Эта функция позволяет среде выполнения более адаптивно использовать JIT-компилятор для достижения лучшей производительности.

Главное достоинство: предоставляет два способа для jit-компиляции методов: в низком-качестве-но-быстро или в более-высоком-качестве-но-медленее. Качество = насколько хорошо оптимизирован метод. Помогает повысить производительность приложения, поскольку оно проходит различные стадии выполнения, от запуска до устойчивого состояния. Когда многоуровневая компиляция отключена, каждый метод компилируется одним способом, который смещен в сторону стабильной производительности по сравнению с производительностью при запуске. (вспоминаем, как работает jit - компиляция)

Когда TC включен, поведение при компиляции матодов при старте следующее:

  • Если у метода есть код посде AOT компиляции или ReadyToRun, то используется предсгенерированный код
  • Иначе метод подвергается jit-компиляции. Обычно эти методы являются универсальными для типов значений.
    • Quick JIT предоставляет менее оптимизированный код быстрее по времи. В .NET Core 3.0 Quick JIT включен по дефолту для методов, не содержащих циклы и предпочтителен для запуска
    • Полностью оптимизированный Jit производит более оптимизированный код более медленно. Для методов, в которых не используется Quick JIT, используется полностью оптимизированный jit.

Для часто вызываемых методов jit компилятор в конечном итоге создает полностью оптимизированный код в фоновом режиме. Затем оптимизированный код заменяет предварительно скомпилированный код для этого метода.
