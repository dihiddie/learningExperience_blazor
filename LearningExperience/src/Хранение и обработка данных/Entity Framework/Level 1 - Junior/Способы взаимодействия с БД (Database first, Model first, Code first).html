Code First

Создаем обычный класс с некоторым набором автосвойст. Каждое свойство будет сопоставляться с отдельным столбцом в таблице из бд.

Надо отметить, что Entity Framework при работе с Code First требует определения ключа элемента для создания первичного ключа в таблице в бд. По умолчанию при генерации бд EF в качестве первичных ключей будет рассматривать свойства с именами Id или [Имя_класса]Id (то есть UserId). Если же мы хотим назвать ключевое свойство иначе, то нам нужно будет внести дополнительную логику на c#.

Затем создаем контекст (наследуем от DbContext), где определяем строку соединения,DbSet, который представляет набор сущностей, хранящихся в базе

Затем через обращение к контексту добавляем свою первую запись: db.Users.Add(user1)

Таким же образом можно использовать CodeFirst к существующей базе данных

Model First

  1. Сначала создаем EDM, выбрав Empty EF Designer model
  2. Из Toolbox надо перетащить Entity и задать ему имя и остальные свойства (могут быть как скалярные (int, float, string), так и сложные (объекты)
  3. Ребилд.
  4. Генерируем БД по модели (Generate Database from Model), задаем коннекшен
  5. Готово.

Database First

  1. Нужен провайдер до той бд, с которой работаем
  2. Добавляем EDM, выбрав новый элемент типа ADO.NET Entity Data Model. Выбираем EF Designer from database
  3. Generate from database
  4. Выбираем те таблицы, модели которых хотим видеть в своем решении
  5. Готово.
